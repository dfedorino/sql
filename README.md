# sql

Проект для практики SQL-запросов с автоматическим тестированием решений. Используется H2 Database и JUnit 5 для валидации SQL-решений через Spring JDBC.

## Цели

- Отработка навыков написания сложных SQL-запросов (JOIN, агрегации, рекурсивные CTE, оконные функции)
- Практика на реалистичных схемах баз данных
- Автоматическая проверка корректности решений

## Структура базы данных

Проект содержит четыре набора таблиц:

### Hotels
Таблицы: `Hotel`, `Room`, `Customer`, `Booking` — моделирование системы бронирования отелей.

### Organization
Таблицы: `Departments`, `Roles`, `Employees`, `Projects`, `Tasks` — организационная структура компании с иерархией сотрудников, проектами и задачами.

### Races
Таблицы: `Classes`, `Cars`, `Races`, `Results` — автомобильные гонки с результатами заездов.

### Vehicles
Таблицы: `Vehicle`, `Car`, `Motorcycle`, `Bicycle` — каталог транспортных средств разных типов.

## Реализованные функции

- **Hotels**: 3 задачи — агрегации бронирований, расходы клиентов, классификация отелей по ценовым категориям
- **Organization**: 3 задачи — рекурсивные запросы к иерархии сотрудников, подсчёт подчинённых, анализ проектов и задач
- **Races**: 5 задач — статистика результатов гонок, фильтрация по средним позициям, оконные функции
- **Vehicles**: 2 задачи — объединение данных из разных типов транспорта, поиск по характеристикам

## Требования

- Java 17+
- Maven 3.6+

## Запуск и тестирование

```bash
# Запуск всех тестов
mvn test

# Запуск тестов для конкретного набора задач
mvn test -Dtest=HotelsSolutionTest
mvn test -Dtest=OrganizationSolutionTest
mvn test -Dtest=RacesSolutionTest
mvn test -Dtest=VehiclesSolutionTest
```

## Структура решений

SQL-файлы решений находятся в `src/test/resources/<набор>/task<N>/`. Каждый тест загружает схему и данные, затем выполняет решение из соответствующего файла и сверяет результат с ожидаемым.
